## 🔌 What is JDBC?

**JDBC (Java Database Connectivity)** is Java's standard API for connecting to relational databases. It provides interfaces and classes to:
- Connect to databases
- Execute SQL queries
- Process results
- Handle database transactions

## 🔧 JDBC Implementation in DBConnection.java

```java
public class DBConnection {
    // JDBC URL format: jdbc:mysql://host:port/database?parameters
    private static final String DB_URL =
        "jdbc:mysql://localhost:3306/civic_app?useSSL=false&allowPublicKeyRetrieval=true&serverTimezone=UTC";

    private static final String DB_USER = "civic_user";
    private static final String DB_PASSWORD = "1234";

    private static Connection connection;  // JDBC Connection object

    public static Connection getConnection() throws SQLException {
        if (connection == null || connection.isClosed()) {
            // JDBC DriverManager creates the connection
            connection = DriverManager.getConnection(DB_URL, DB_USER, DB_PASSWORD);
        }
        return connection;
    }
}
```

**JDBC Connection Flow:**
1. **DriverManager.getConnection()** - Establishes database connection
2. **Connection object** - Represents the database session
3. **SQLException** - Handles connection errors

## 🛠️ JDBC Operations in IssueService.java

### 1. INSERT Operation (Creating Issues):
```java
public boolean addIssue(Issue issue) {
    String sql = "INSERT INTO issues (category, description, location, status, image_path) VALUES (?, ?, ?, ?, ?)";

    try (Connection conn = DBConnection.getConnection();
         PreparedStatement stmt = conn.prepareStatement(sql)) {

        // JDBC PreparedStatement parameter binding
        stmt.setString(1, issue.getCategory());
        stmt.setString(2, issue.getDescription());
        stmt.setString(3, issue.getLocation());
        stmt.setString(4, issue.getStatus());
        stmt.setString(5, issue.getImagePath());

        // JDBC executeUpdate() for INSERT/UPDATE/DELETE
        return stmt.executeUpdate() == 1;

    } catch (SQLException ex) {
        ex.printStackTrace();
        return false;
    }
}
```

### 2. SELECT Operation (Reading Issues):
```java
public List<Issue> getAllIssues() {
    String sql = "SELECT * FROM issues ORDER BY created_at DESC";
    List<Issue> issues = new ArrayList<>();

    try (Connection conn = DBConnection.getConnection();
         PreparedStatement stmt = conn.prepareStatement(sql);
         ResultSet rs = stmt.executeQuery()) {  // JDBC executeQuery() for SELECT

        // JDBC ResultSet processing
        while (rs.next()) {
            issues.add(mapIssue(rs));  // Convert ResultSet row to Issue object
        }
    } catch (SQLException ex) {
        ex.printStackTrace();
    }
    return issues;
}
```

### 3. UPDATE Operation (Status Changes):
```java
public boolean updateIssueStatus(int id, String newStatus, String proofImagePath) {
    String sql = "UPDATE issues SET status = ?, updated_at = CURRENT_TIMESTAMP WHERE id = ?";

    try (Connection conn = DBConnection.getConnection();
         PreparedStatement stmt = conn.prepareStatement(sql)) {

        stmt.setString(1, newStatus);
        stmt.setInt(2, id);

        return stmt.executeUpdate() == 1;  // Returns affected rows count

    } catch (SQLException ex) {
        ex.printStackTrace();
        return false;
    }
}
```

        rs.getTimestamp("updated_at")     // JDBC: getTimestamp(columnName)

## 🛡️ JDBC Security Features Used

### 1. PreparedStatement (SQL Injection Prevention):
```java
// SAFE: Parameters are bound separately from SQL
PreparedStatement stmt = conn.prepareStatement(
    "SELECT * FROM issues WHERE location LIKE ? AND status = ?"
);
stmt.setString(1, "%" + location + "%");  // Parameter binding
stmt.setString(2, status);

// UNSAFE (if using Statement):
// String sql = "SELECT * FROM issues WHERE location LIKE '%" + location + "%'";
```

### 2. Parameter Binding Methods:
- **setString()** - For VARCHAR/TEXT columns
- **setInt()** - For INTEGER columns
- **setTimestamp()** - For TIMESTAMP columns
- **setObject()** - Generic parameter binding

## 🔄 JDBC Resource Management

### Try-with-Resources (Automatic Cleanup):
```java
try (Connection conn = DBConnection.getConnection();
     PreparedStatement stmt = conn.prepareStatement(sql);
     ResultSet rs = stmt.executeQuery()) {
    // JDBC resources automatically closed here
} catch (SQLException ex) {
    // Handle exceptions
}
```

**Why this matters:**
- **Connection leaks** prevented
- **Memory leaks** prevented
- **Database connection pool** efficiency
- **Resource exhaustion** prevented

## 🔄 JDBC Transaction Management

### Implicit Transactions (Used in Project):
- Each JDBC operation is auto-committed
- Simple for this desktop application
- Suitable for single-user scenarios

### Production Transaction Pattern:
```java
conn.setAutoCommit(false);  // Start transaction
try {
    // Multiple operations
    stmt1.executeUpdate();
    stmt2.executeUpdate();
    conn.commit();          // Commit all changes
} catch (SQLException ex) {
    conn.rollback();        // Rollback on error
} finally {
    conn.setAutoCommit(true); // Restore default
}
```
